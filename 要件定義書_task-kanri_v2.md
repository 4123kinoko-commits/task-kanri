# 要件定義書

**プロジェクト名：** task-kanri（Trello風タスク管理アプリ）  
**作成日：** 2026年6月5日  
**バージョン：** 2.0

---

## 1. プロジェクト概要

| 項目 | 内容 |
|------|------|
| プロジェクト名 | task-kanri |
| 目的 | タスクをカンバン形式で視覚的に管理する |
| 対象ユーザー | 個人利用（自分一人） |
| 開発環境 | Cursor / Windows |

---

## 2. 技術スタック

### 2-1. フロントエンド（画面側）

| 項目 | 技術 | 役割 |
|------|------|------|
| フレームワーク | React | 画面のパーツを部品として管理する |
| ビルドツール | Vite | 開発を高速化するツール |
| スタイリング | Tailwind CSS | クラス名を書くだけで見た目を整えられるCSS |
| ドラッグ＆ドロップ | dnd-kit | カードをドラッグで移動させる機能 |

### 2-2. バックエンド（サーバー側）

| 項目 | 技術 | 役割 |
|------|------|------|
| 言語 | Node.js | JavaScriptをサーバーで動かす仕組み |
| フレームワーク | Express | サーバーの処理をシンプルに書けるツール |
| API形式 | REST API | フロントとバックエンドがデータをやり取りする方法 |

### 2-3. データベース

| 項目 | 技術 | 役割 |
|------|------|------|
| RDBMS | PostgreSQL | タスクデータを保存・管理するデータベース |

### 2-4. 開発ツール

| 項目 | 技術 | 役割 |
|------|------|------|
| バージョン管理 | Git + GitHub | コードの変更履歴を管理・保存する |

---

## 3. 機能要件

機能要件とは「このアプリで何ができるか」を定義するものです。

### 3-1. カード機能

| No. | 機能 | 説明 |
|-----|------|------|
| F-01 | カード追加 | 各列にカードを新規追加できる |
| F-02 | カード削除 | 不要なカードを削除できる |
| F-03 | タイトル入力 | カードにタイトルを入力できる |
| F-04 | 優先度設定 | カードに優先度（高・中・低）を設定できる |
| F-05 | カード移動 | ドラッグ＆ドロップで列間・列内を移動できる |

### 3-2. 列（カラム）機能

| No. | 機能 | 説明 |
|-----|------|------|
| F-06 | 列追加 | 任意のタイトルで新しい列を追加できる |
| F-07 | 列削除 | 不要な列を削除できる |
| F-08 | 列タイトル設定 | 列に任意のタイトルをつけられる |

### 3-3. データ保存機能

| No. | 機能 | 説明 |
|-----|------|------|
| F-09 | 自動保存 | 操作のたびにデータベースへ自動保存される |
| F-10 | データ復元 | ブラウザを閉じて再度開いてもデータが復元される |

---

## 4. 非機能要件

非機能要件とは「どんな品質・性能であるべきか」を定義するものです。

| No. | 項目 | 内容 | 理由 |
|-----|------|------|------|
| N-01 | レスポンシブ対応 | PC・スマホ両対応のレイアウト | どの端末からでも利用できるようにするため |
| N-02 | 表示速度 | 操作後1秒以内に画面に反映される | 快適に操作できるようにするため |
| N-03 | データ保持 | ブラウザを閉じてもデータが消えない | 作成したタスクが失われないようにするため |
| N-04 | 直感的なUI | 説明なしで操作方法が理解できるデザイン | 誰でもすぐ使い始められるようにするため |
| N-05 | 可読性 | 小さい画面でも読みやすい文字サイズ・コントラスト | スマホでも視認性を確保するため |

## 8. データの流れ

```
ブラウザ（React）
　　↕ REST API（HTTP通信）
サーバー（Node.js + Express）
　　↕ SQL
データベース（PostgreSQL）
```

---

## 9. 対応しないこと（スコープ外）

- ログイン・ユーザー管理
- 複数ユーザーでの共有
- 締め切り日・担当者名の設定
- メモ・説明文の入力
- スマホでのドラッグ＆ドロップ操作（見た目のスマホ対応のみ実施）

## 10. 用語集

| 用語 | 説明 |
|------|------|
| React | 画面をパーツ（コンポーネント）に分けて作るJavaScriptの仕組み |
| Vite | Reactアプリを素早く起動・開発できるツール |
| Tailwind CSS | クラス名を書くだけでデザインできるCSSの仕組み |
| dnd-kit | ドラッグ＆ドロップ機能を簡単に実装できるライブラリ |
| Node.js | JavaScriptをサーバー側で動かす仕組み |
| Express | Node.jsでサーバーを簡単に作れるフレームワーク |
| REST API | フロントとバックエンドがデータをやり取りするルール |
| PostgreSQL | データを表形式で管理するデータベース |
| カンバンボード | タスクを列ごとに視覚的に管理する手法。Trelloが代表例 |
| コンポーネント | 画面を構成する再利用可能なパーツのこと |
